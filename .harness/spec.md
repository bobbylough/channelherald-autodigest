## Summary

Build the `channelherald-autodigest` pipeline from scratch: a Python automation
that fetches newsletters via IMAP, scores and scrapes linked articles, rates and
summarises them via LLM, deduplicates by topic, and sends a curated HTML digest
by email. The spec covers all 12 modules needed to take an empty repo to a
working pipeline. No existing source code to modify — every file is net-new.

---

## Constraints

- Python 3.11+ — no walrus operator overuse, no magic
- No external dependencies beyond: `beautifulsoup4`, `requests`, `imaplib` (stdlib), `smtplib` (stdlib), `sqlite3` (stdlib) — add to `pyproject.toml` as needed
- Every task must have at least one failing test before any implementation (TDD)
- Coverage must stay at or above 90% after each task
- KISS: simplest solution that passes the test
- YAGNI: nothing built beyond what the current task requires
- Functions ≤ 20 lines, one responsibility each
